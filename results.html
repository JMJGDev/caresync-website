The respondents performed tasks using the prototype and answered questions about their experience. Their responses helped identify what works well and what needs improvement in the system.

DISCUSSION OF FINDINGS

Key Findings

Analysis of both quantitative metrics and qualitative feedback revealed consistent patterns in user behavior and recurring pain points.

🛠️ Usability Issues Identified

HIGH SEVERITY 🐢

Slow System Response / Lag

Multiple users reported the system was slow to respond when tapping elements, creating frustration and uncertainty about whether actions were registered.

HIGH SEVERITY 📘

Missing Onboarding / Instructions

Users expressed confusion about how to navigate the system. No tutorial or onboarding flow was present, leading to exploratory and error-prone behavior.

MEDIUM SEVERITY 🌐

Language Accessibility

A user flagged that not all users are English-proficient. No language preference or translation feature is available, potentially excluding non-student users.

MEDIUM SEVERITY 🔘

Button Interaction Issues

Some users noted the click/tap buttons were not intuitive or needed redesign, suggesting affordance and visual feedback improvements are needed.

LOW SEVERITY 📢

Low System Awareness / Visibility

One user suggested promoting the system more widely, indicating low discoverability and lack of user base awareness outside the study group.

LOW SEVERITY 🎨

Visual Design Clarity

A participant mentioned the interface is "a little confusing," pointing to layout or visual hierarchy issues that may impede initial orientation.

RECOMMENDATIONS

Design Recommendations

Based on usability issues, SUS scores, and open-ended feedback, the following improvements are recommended to enhance the CareSync prototype experience.

1

Optimize System Performance & Response Time

Address the lag and slow tap response reported by multiple participants. Implement loading indicators, optimize API calls, and use local caching for clinic data to ensure the system responds within 1–2 seconds of user interaction.

HIGH PRIORITY
2

Add an Onboarding / Tutorial Flow

Introduce a brief onboarding sequence (3–5 screens) that guides first-time users through core features. Include tooltips and contextual hints to reduce initial confusion, particularly for navigating the clinic finder and map features.

HIGH PRIORITY
3

Improve Button Affordance & Visual Feedback

Redesign interactive elements to clearly communicate clickability. Add tap animations, ripple effects, and state changes (hover, active, loading) so users receive immediate visual confirmation of their actions.

HIGH PRIORITY
4

Introduce Multi-Language Support

Add a language preference option (at minimum, Filipino/English toggle) to broaden accessibility beyond student populations. Consider localized labels for medical terms to serve non-academic users effectively.

MEDIUM PRIORITY
5

Simplify Interface Layout & Visual Hierarchy

Reduce visual clutter on key screens. Apply clearer typography hierarchy, consistent spacing, and logical grouping of related elements to help users orient themselves quickly without external guidance.

MEDIUM PRIORITY
6

Conduct Follow-Up Usability Testing

After implementing the above improvements, repeat usability testing with the same or similar participant group to measure progress and verify that the SUS score improves beyond the "Marginal" range toward "Good" (≥ 68).

LOW PRIORITY
